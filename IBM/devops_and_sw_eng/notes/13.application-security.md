## Application Security for Developers and DevOps Professionals

### Introduction to DevSecOps

- The secure software development lifecycle (SDLC) is a process that involves security testing and its best practices in the existing development model.
- SDLC stages are requirements, design, development, testing, and deployment. You can map secure SDLC to these stages by undertaking risk assessments and looking at how individuals may attack your code.
- DevSecOps automates security integration throughout the concept, implementation, testing, deployment, and delivery SDLC process.
- DevSecOps provides higher-quality software, improved security, vulnerability fixings, and faster recovery after a security incident.

### Role of Network Security

- OSI model is a system that allows for present and future types of communication to be used by everyone globally.
- The OSI model consists of seven layers for sending and receiving data. The first layer is the physical layer, then the data link layer, the network layer, the transport layer, the session layer, the presentation layer, and finally the application layer.
- Securing the communications layer is important to manage using SSH, HTTPS, SSL/TLS, or the secure sockets layer.
- When application developers need to commit code to an application project, the security code delivery pipeline layer should be secured and restricted.
- Security patterns are a set of rules that represent and define a reusable solution to recurring security threats or issues.
- Transport Layer Security (or TLS) and Secure Sockets Layer (or SSL) are protocols for establishing secure connections or communications between networked computers, specifically a server and a client.
- TLS works effectively when the server has an up-to-date certificate and TLS version support.
- OpenSSL is an open-source toolkit that uses a command line and software libraries to ensure secure communication with cryptography for all types of communication, from personal to commercial and e-commerce transactions.

### Inspecting Security in Application Development

- Threat monitoring is scanning code repositories and containers to find security issues.
- Integrating threat monitoring with repositories enables a code checker to scan source code for security issues.
- Authentication verifies a user’s identity, and authorization determines a user’s access rights.
- Symmetric encryption means encrypting and decrypting with the same keys, whereas asymmetric encryption means encrypting and decrypting with different keys.

### Security Testing and Mitigation Strategies

- Using mitigation strategies helps reduce risks and impacts of security threats and vulnerabilities.
- Static analysis automatically inspects source code before execution.
- Static code analysis tools offer depth, speed, and accuracy.
- You perform dynamic analysis after you deploy the code to production.
- Dynamic testing helps detect faults in dynamic code pathways that other tests may overlook.

### Key Analysis in Applications

- Secure code review focuses on threats that are exposed or source code that contains security-critical components.
- Automated code reviews evaluate large codebases quickly and efficiently.
- Three vulnerability tools you can use are Burp Suite, Nessus, and ZAP.
- Training, policy updating, and process automation are best practices to follow in vulnerability analysis.
- Runtime protection is a security mechanism that protects your applications against vulnerabilities while they are running.
- The goal of software component analysis (SCA) is to discover open-source components in a codebase so that you can better manage security and license compliance risks.
- Continuous security analysis reduces risks, removes bottlenecks in software development, and accelerates new feature development.

### Introducing OWASP Top 10

- The Open Web Application Security Project (OWASP) Top 10 is a report that identifies current software security vulnerability concerns.
- The five major steps in developing the OWASP Top 10 are to plan the schedule and call for data, survey the industry, analyze the data, draft the list with categories from the data and the survey for review, and after reaching consensus, release the list.
- Hackers exploiting access control compromise application security, tarnish company reputation, cause financial losses.
- Hackers can easily decrypt common encryption methods, so you should encrypt sensitive data in databased and in transit.
- Insecure design is a lack of security, security misconfiguration consists of overlooked weaknesses, and vulnerable and outdated components make your applications more vulnerable to attacks.
- Using security measures and staying informed of security updates can protect your applications.
- Use secure application designs, robust logging and monitoring, and secure web servers to prevent SSRF attacks to protect your applications and infrastructure.
- Integrity checks prevent tampering, web servers should be configured correctly, input data should be sanitized, and unsigned data should never be sent to untrusted clients.

### Diving Deeper into OWASP

- SQL injection is an attack that passes a string input to exploit a database.
- Four types of structured query language (SQL) injection are: SQL manipulation, code injection, function call injection, and buffer overflow.
- You can protect your application against SQL injection attacks by using query parameters, validating on the server side, restricting user privileges, and performing dynamic application security testing (DAST).
- Cross-site scripting is when an application sends untrusted data to a browser, and the three common types are stored, blind, and reflected.
- You can prevent cross-site scripting by looking for suspicious HTTP requests and keywords, escaping lists and keywords, turning off HTTP TRACE, and avoiding unsafe sinks.
- Secrets management is storing and managing items like passwords that must be kept secret.
- Three common methods to store and manage secrets are the graphical user interface (GUI), command line interface (CLI), and hypertext transport protocol application programming interface (HTTP API).
- Challenges in developing code include accessibility, auditing and logging, and security.

### Code Development Practices

- Code practices help mitigate vulnerabilities, and including security early in the software development lifecycle saves money.
- Using trusted code reduces risk, and meaningful, secure error handling is the goal.
- A dependency is a code that another code relies on and can add features and functionality to your application.
- Dependencies speed up development, but they should be thoroughly vetted before use.
- A secure development environment includes collaboration with security teams, which helps you with defensive code development.
- An insecure development environment includes compromised development machines, a lack of protective monitoring, logging, and auditing, and allows unrestricted access to machines and code repositories.
